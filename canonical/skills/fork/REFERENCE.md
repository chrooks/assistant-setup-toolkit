# /fork — Reference

Companion to [SKILL.md](SKILL.md). Holds detail kept out of the core to stay under the 100-line skill budget.

## Slug rules

Kebab-case, alphanumeric + dashes, max 60 chars. Inferred from conversation context; user can override at fork time. Surface the inferred slug *with* the override prompt.

Collision behavior:

- **Handoffs** — collision-free (timestamp prefix `<timestamp>-<slug>.md`).
- **ADRs** — collision-free (sequence prefix `####-slug.md`).
- **PRDs** — always prompt: *"PRD `<slug>` exists. Update / save as `<slug>-v2` / pick new slug?"* Never silently overwrite — would split a source-of-truth document (ADR-0001).
- **Artifacts** — prompt unless update intent is clear ("update the budget spreadsheet"); otherwise auto-suffix `-2`. Default to prompt when ambiguous.

## Path conventions

User-overridable via `.cowork/config.yaml`:

| Kind | Default path | Config key |
|------|--------------|------------|
| Glossary | `CONTEXT.md` (root) | — |
| ADRs | `docs/adr/####-slug.md` | `adr_dir` |
| PRDs | `docs/prd/<slug>.md` | `prd_dir` |
| Artifacts | `artifacts/<slug>.<ext>` | `artifacts_dir` |

Non-configurable Cowork plumbing (treat like `.git/`):

- Handoffs — `.cowork/handoffs/<timestamp>-<slug>.md`
- Scratch — `.cowork/scratch/`
- Index — `.cowork/index.md`
- Config — `.cowork/config.yaml`

## `.cowork/config.yaml` schema (v1)

Optional file. Absence = all defaults.

| Key | Values | Default |
|-----|--------|---------|
| `run_grill_with_docs` | `always` \| `ask` \| `never` | `ask` |
| `run_to_prd` | `always` \| `ask` \| `never` | `ask` |
| `adr_dir` | path | `docs/adr/` |
| `prd_dir` | path | `docs/prd/` |
| `artifacts_dir` | path | `artifacts/` |

`run_*` keys are written after the first-fork prompt unless user picked "ask each time." Schema grows lazily — only when a real consumer needs a key.

## Handoff format

**Hybrid** — embedded short summary (last decision, open questions, next concrete step) + path references to canonical `CONTEXT.md` and PRD. New session reads embed for immediate context, pulls depth from referenced files.

**Mode 3 degrades to fully-self-contained** — full text embedded, no path references (no on-disk files to point at).

## `.cowork/index.md` format

Plain markdown, regenerated by `/fork` only. Three sections: source-of-truth docs, artifacts, handoffs. Header line shows last-update time + originating fork so staleness is visible. No frontmatter until a real machine consumer needs it (purely-additive upgrade).

Example:

```markdown
# Project Index

Last updated: 2026-05-06 by /fork (handoff: 2026-05-06-routing-design)

## Source-of-truth documents
- [CONTEXT.md](../CONTEXT.md) — domain glossary
- [docs/prd/fork-skill.md](../docs/prd/fork-skill.md) — /fork PRD

## Artifacts
(none yet)

## Handoffs
- [2026-05-06-routing-design.md](handoffs/2026-05-06-routing-design.md) — design grilling
```

## Failure modes

- **No filesystem, user wants a folder** — explain Mode 2 needs Cowork or a filesystem MCP; offer Mode 3.
- **PRD slug collision** — always prompt, never silently overwrite.
- **`mcp__cowork__list_artifacts` empty in Mode 1** — silently skip widget-promotion step.
- **`.cowork/config.yaml` malformed** — log error, fall back to defaults, prompt user to repair.
- **Existing convention drift** (e.g. user has `docs/architecture/decisions/` but no config) — known v1 limitation; future: detect on first run and prompt for config write.

## Mode 3 — Chat fallback

Activated when no filesystem is available, or user declines Mode 2. All output emitted directly in chat as fenced markdown blocks. No files written anywhere.

### Step-by-step behavior in Mode 3

| Step | Mode 1/2 behavior | Mode 3 behavior |
|------|-------------------|-----------------|
| 1 | Detect folder/filesystem | Detect no filesystem → announce Mode 3 |
| 2 | Read `.cowork/index.md` | Skip — no index on disk. Print *"Mode 3: no project folder. Artifacts emitted in chat."* |
| 3 | Read/create `.cowork/config.yaml` | Skip — no config file. Use defaults for prep-step decisions (`ask` for both). |
| 4 | Optional `grill-with-docs` | Still prompt per default `ask`. If user says yes, run `grill-with-docs` but instruct it to **emit CONTEXT.md content in chat** instead of writing to disk. |
| 5 | Optional `to-prd` | Same — run `to-prd` in emit-to-chat mode. PRD content printed as fenced markdown. |
| 6 | Sweep `.cowork/scratch/` | Skip — no scratch dir. |
| 7 | Widget promotion | Skip — no filesystem to export to. |
| 8 | Handoff write | **8e applies.** Skip file write. Emit fully self-contained handoff (no path references). Inline the CONTEXT.md and PRD content produced by Steps 4–5 into the handoff body so the next session has everything in one block. |
| 9 | Index regen | Skip — no filesystem. |

### Prep-skill chat emission

When `grill-with-docs` or `to-prd` runs in Mode 3, the orchestrator instructs them to return their output as a string rather than writing files. The orchestrator then prints each as a labeled fenced block:

```
📋 CONTEXT.md (would be written to project root):
\`\`\`markdown
<content>
\`\`\`

📋 PRD — <slug> (would be written to docs/prd/<slug>.md):
\`\`\`markdown
<content>
\`\`\`
```

The user can copy these into their project manually. The handoff (Step 8e) also embeds this content so the next session has it without manual file creation.

### Self-contained handoff format (Mode 3)

The Mode 3 handoff differs from the hybrid format (8d) in two ways:

1. **No path-reference header** — no `<!-- CONTEXT: ... -->` or `<!-- PRD: ... -->` lines (no files to point at).
2. **Embedded context sections** — after the handoff body, append:

```markdown
---
## Embedded context (no project folder — copy to files if desired)

### CONTEXT.md
<full CONTEXT.md content from Step 4, or "not generated">

### PRD
<full PRD content from Step 5, or "not generated">
```

This makes the handoff fully self-contained: paste it into a new session and all context is inline.

## Mode 2 — Materialize project folder

Activated when Step 1 detects no project folder (no `CONTEXT.md`, `.cowork/`, `docs/`, `.git/`, `package.json`, `pyproject.toml` in cwd) **but** filesystem access is available (either `mcp__cowork__request_cowork_directory` tool exists, or running in Claude Code / connected filesystem MCP).

### Detection

After Mode 1 check fails, probe for filesystem capability:
1. Check if `mcp__cowork__request_cowork_directory` tool is available (Cowork environment).
2. Otherwise, check if Write/Bash tools can create files in cwd (Claude Code / filesystem MCP).
3. If neither → fall through to Mode 3.

### Prompt

Present three options:

> *No project folder detected. Options:*
> 1. **Create here** — scaffold in current directory (`<cwd>/`)
> 2. **Create at path** — scaffold at a custom path (enter path)
> 3. **Skip** — continue without a folder (Mode 3)
>
> *Recommended: 1 (create here)*

In Cowork, option 1 uses `mcp__cowork__request_cowork_directory` to request a folder if one isn't already mounted. In Claude Code, option 1 scaffolds directly in cwd.

If user picks option 3 or cancels → fall through to Mode 3.

### Scaffold layout

Create the minimal directory structure per the CONTEXT.md directory layout:

```
<target>/
├── CONTEXT.md                  # empty placeholder with header
├── docs/
│   ├── adr/                    # empty dir
│   └── prd/                    # empty dir
├── artifacts/                  # empty dir
└── .cowork/
    ├── handoffs/               # empty dir
    ├── scratch/                # empty dir
    └── index.md                # initial empty index
```

**CONTEXT.md placeholder content:**

```markdown
# CONTEXT

Domain glossary for this project. Populated by `/fork` → `grill-with-docs`.
```

**`.cowork/index.md` initial content:**

```markdown
# Project Index

Last updated: <YYYY-MM-DD> by /fork (initial scaffold)

## Source-of-truth documents
- [CONTEXT.md](../CONTEXT.md) — domain glossary (placeholder)

## Artifacts
(none yet)

## Handoffs
(none yet)
```

### Post-scaffold

After scaffolding:
1. Print confirmation: *"Project folder scaffolded at `<path>/`. Continuing as Mode 1."*
2. Set mode to Mode 1 and continue with Step 2 (pre-fork index read).

All subsequent steps run as normal Mode 1 — config prompt (Step 3), prep steps (4–5), scratch sweep (6), handoff (8), index regen (9).

### Config path overrides

If user picked option 2 (custom path), the scaffolded path becomes the working directory for the rest of the fork. All path-relative operations (config read, handoff write, index regen) use this path as root.

If `.cowork/config.yaml` path overrides exist (e.g. from a prior fork), respect them. On fresh scaffold, no config file exists yet — Step 3 handles first-fork config creation as normal.

## Scratch-by-intent rule (ADR-0004)

Skills route writes by **intent**, not file type:

- Clear deliverable request ("make me the budget spreadsheet") → write directly to `artifacts/`.
- Iterative/exploratory work ("let me try a few approaches") → write to `.cowork/scratch/`.
- Ambiguous → default to scratch. `/fork`'s sweep is the safety net.

## Step 7 — Promote Cowork HTML widgets

Runs after scratch sweep (Step 6), before handoff write (Step 8). Cowork-only — exports HTML widgets from Cowork's artifact registry to project files.

### Guard

Check if `mcp__cowork__list_artifacts` tool is available. If not (Claude Code, non-Cowork environment), skip silently. Print nothing.

### List

Call `mcp__cowork__list_artifacts` to get widgets created during the conversation. If empty, skip silently.

Print a numbered list:

> *Cowork widgets from this session:*
> 1. `pricing-calculator` — interactive pricing table (widget ID: `abc123`)
> 2. `org-chart` — team structure visualization (widget ID: `def456`)

### Prompt

> *Promote to `artifacts/`? Enter numbers (e.g. `1,2`), `all`, or `none` to skip.*

Default recommendation: `none` (widgets are often exploratory).

### Export (snapshot mode)

For each selected widget:

1. Ensure `artifacts/` dir exists (respect `artifacts_dir` config override).
2. Render the widget headlessly. Allow `window.cowork.callMcpTool` calls to resolve. Capture post-hydration DOM.
3. Save as `artifacts/<slug>.html`. If collision, prompt: *"`<slug>.html` exists. Overwrite / rename to `<slug>-2.html` / skip?"*
4. Write companion `artifacts/<slug>.md` stub:

```markdown
# <Widget title>

- **Source:** Cowork widget `<widget-id>`
- **Snapshot date:** <YYYY-MM-DD>
- **Description:** <one-sentence description from registry metadata>

> This is a static snapshot. The original widget may have interactive features not preserved here.
```

### Summary

Print: *"Promoted N widget(s), skipped M."*

### Limitations

**Snapshot mode only** in v1. Source-mode export (preserving live `cowork.callMcpTool` calls for re-import) is Phase 4. Snapshotted HTML is static — interactive features that depend on MCP tool calls won't function in the exported file.

## Step 3 — Config read

Read `.cowork/config.yaml` if it exists. Parse keys per the schema above (§ `.cowork/config.yaml` schema). If the file is missing or malformed, fall back to all defaults and continue.

**First-fork behavior:** If `.cowork/config.yaml` does not exist, prompt the user for `run_grill_with_docs` and `run_to_prd` preferences:

> *"First fork in this project. Should `/fork` run these prep steps?"*
> - `grill-with-docs`: always / ask each time / never
> - `to-prd`: always / ask each time / never

Write answers to `.cowork/config.yaml`. If user picks "ask each time," write `ask` (the default).

## Step 4 — Optional `grill-with-docs`

Decision logic using `run_grill_with_docs` from config:

- `always` → invoke `grill-with-docs` unconditionally.
- `ask` → recommend based on heuristics, then prompt:
  - Recommend **yes** if: no `CONTEXT.md` exists, OR conversation has substantive domain content (new terms, design decisions, trade-offs discussed).
  - Recommend **skip** if: `CONTEXT.md` exists and conversation was narrow/tactical.
  - Prompt: *"Run grill-with-docs? (Recommended: <yes|skip>) [Y/n]"*
- `never` → skip silently.

When invoked, `grill-with-docs` may create or update `CONTEXT.md` and files in `docs/adr/`. These are inputs to Step 8 (handoff) and Step 9 (index regen).

## Step 5 — Optional `to-prd`

Decision logic using `run_to_prd` from config:

- `always` → invoke `to-prd` unconditionally.
- `ask` → recommend based on heuristics, then prompt:
  - Recommend **yes** if: conversation has converged on design conclusions and no matching PRD exists in `docs/prd/`.
  - Recommend **skip** if: conversation was exploratory/early, or a PRD already exists for this topic.
  - Prompt: *"Run to-prd? (Recommended: <yes|skip>) [Y/n]"*
- `never` → skip silently.

When invoked, `to-prd` writes `docs/prd/<slug>.md` (per ADR-0001 dual-write). The slug is inferred from conversation context; collision handling per § Slug rules above.

## Step 6 — Sweep `.cowork/scratch/`

Runs after prep steps (4–5), before handoff write (8). Promotes scratch files the user wants to keep; discards the rest.

### Guard

If `.cowork/scratch/` does not exist or is empty, skip silently. Print nothing.

### List

List all files in `.cowork/scratch/` (non-recursive, flat listing). Print a numbered list with filenames and sizes:

> *Scratch files from this session:*
> 1. `draft-api-spec.md` (2.1 KB)
> 2. `color-palette-experiment.html` (4.8 KB)
> 3. `notes.txt` (0.3 KB)

### Prompt

Ask user which files to promote:

> *Promote to `artifacts/`? Enter numbers (e.g. `1,3`), `all`, or `none` to discard everything.*

Default recommendation: `none` (scratch is disposable by design per ADR-0004).

### Promote

For each selected file:
1. Ensure `artifacts/` dir exists (respect `artifacts_dir` config override from `.cowork/config.yaml`).
2. Move (not copy) the file to `artifacts/<filename>`. If a file with the same name already exists in `artifacts/`, prompt: *"`<filename>` already exists in artifacts. Overwrite / rename to `<filename>-2` / skip?"*
3. Print confirmation: `Promoted: <filename> → artifacts/<filename>`.

### Cleanup

After promotion decisions, delete remaining (non-promoted) files from `.cowork/scratch/`. Do **not** remove the directory itself — it may be reused.

Print summary: *"Promoted N file(s), discarded M."*

## Step 10 — Rich surface

Runs after Step 9 (index regen). Surfaces the handoff to the user with the best available affordance.

### Basic surface (current, always available)

Already implemented in Step 8f:
1. Path to written handoff file (Mode 1/2).
2. Handoff content as fenced code block for copy-paste.
3. Note on deferred steps.

### Rich surface (when available)

If the environment supports a "start new chat with seed" affordance (e.g. Cowork's `mcp__cowork__start_new_chat` or similar), offer it:

> *Start a new session with this handoff pre-loaded? [Y/n]*

If accepted, invoke the affordance with the handoff content as the seed message. If the tool is unavailable or user declines, fall back to basic surface.

### Detection

Check for known seed-chat tools in order:
1. `mcp__cowork__start_new_chat` (Cowork)
2. Future: IDE-specific tools (VS Code extension APIs, etc.)

If none found, basic surface only. No error — this is purely additive.

## Existing-skill extensions (gentle, not rewrite)

Each of `grill-with-docs`, `to-prd`, `handoff` keeps its current trigger, behavior, and standalone usefulness. Two uniform additions when running inside a project folder:

### 1. Read `.cowork/config.yaml` for path overrides

Before writing any file, check if `.cowork/config.yaml` exists and read relevant path keys:

| Skill | Config keys used |
|-------|-----------------|
| `grill-with-docs` | `adr_dir` (default: `docs/adr/`) |
| `to-prd` | `prd_dir` (default: `docs/prd/`) — **already implemented** |
| `handoff` | none (handoff path `.cowork/handoffs/` is not configurable) |

If config file missing or key absent, use defaults. If malformed, log warning and use defaults.

### 2. Update `.cowork/index.md` on standalone invocation

After writing files, check if `.cowork/index.md` exists. If so, update the relevant section:

| Skill | Index section updated |
|-------|----------------------|
| `grill-with-docs` | **Source-of-truth documents** — add/update `CONTEXT.md` entry, add new ADR entries |
| `to-prd` | **Source-of-truth documents** — add new PRD entry |
| `handoff` | **Handoffs** — add new handoff entry |

Update the header timestamp: `Last updated: <YYYY-MM-DD> by /<skill-name>`.

If `.cowork/index.md` does not exist, skip — no-op. Running standalone outside a project folder works unchanged.

### Backwards compatibility

These additions are **no-ops** when `.cowork/` doesn't exist. Standalone skill invocations outside `/fork` continue working exactly as before. The extensions only activate when project folder conventions are present.

Skill-specific: `to-prd` **dual-writes** `docs/prd/<slug>.md` to the project folder in addition to any tracker publishing (ADR-0001). Local file is canonical; tracker copy is for human collaboration.
