# Visual Picker — Concept-Shape → Form

Reach for a visual when a concept has a **shape a visual carries** — not because someone
is "a visual learner." Words + a *relevant* visual help everyone (dual-coding); a
decorative one only costs attention. Match the form to the concept's shape, then render it.

## Cheat Sheet (concept shape → form)

| Concept shape | Best form |
|---|---|
| Multi-attribute comparison (options × dimensions), or looking up exact values | **Table** — `/table` |
| Quantitative *shape* — a trend over time, magnitude comparison, ranking, distribution, correlation | **Chart** — `/figure` |
| Process / pipeline / linear steps | **Numbered list** (linear) or **flowchart** (branches/loops) — `/diagram` |
| Architecture / how parts connect | **Node-and-edge diagram** — `/diagram` |
| State machine / lifecycle | **State diagram** — `/diagram` |
| Hierarchy / part-whole / containment | **Tree** (inline ASCII; treemap if quantitative) |
| Short set of parallel/sequential points | **Bulleted / numbered list** |
| Genuine narrative, nuance, "why," trade-off reasoning | **Prose** (no visual) |

## Guardrails

- **Representational, not decorative.** A visual must carry the structure. Decorative or
  redundant visuals add load without paying it back — skip them.
- **Integrate labels into the visual.** Don't make the reader bounce between a prose wall
  and a separate unlabeled figure (avoid split-attention).
- **Introduce the figure in the sentence before it.** Name what the table or diagram shows
  right before it lands.
- **If a table cell needs more than ~2 sentences, it belongs in another form** — that cell
  is prose or its own diagram, not a table cell.
- **Light expertise taper.** Lean on the visual when the material is new or complex; taper
  it as the reader owns the model (over-scaffolding an owned topic backfires).
- **Skip visuals for nuance, narrative, or "why."** Those are prose by shape.
